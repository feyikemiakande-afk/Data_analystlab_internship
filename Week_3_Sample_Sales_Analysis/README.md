# 📊 Week 3: Sample Sales SQL Analysis

## 📌 Project Overview

This project was completed as part of **Week 3** of the **AnalystLab Africa Data Analytics Internship**. The objective was to strengthen SQL skills by analysing a transactional sales dataset and applying SQL techniques to answer business questions and generate meaningful business insights.

The project involved database exploration, data cleaning, SQL querying, business analysis, and query optimisation using MySQL.

---

## 🎯 Objectives

- Explore and understand a transactional sales dataset.
- Clean and prepare the data for analysis.
- Retrieve and analyse data using SQL.
- Apply advanced SQL concepts to solve business problems.
- Optimise query performance using indexes and execution plans.

---

## 🗂️ Dataset Overview

The Sample Sales dataset contains transactional sales information, including:

- Customer Details
- Product Information
- Sales Values
- Order Details
- Cities and Countries
- Deal Sizes
- Sales Periods (Year and Month)

The dataset was analysed to identify sales trends, customer behaviour, and top-performing products.

## 💼 Business Questions Answered

This project answers several business questions, including:

- Which product lines generated the highest revenue?
- Who are the highest-spending customers?
- How has revenue changed over time?
- Which cities generated the highest sales?
- How do customers differ in purchasing behaviour?
- Which customers spent more than the specified sales threshold?

## 📈 Key Insights

- Product lines contributed differently to overall revenue.
- A small group of customers generated a significant proportion of total sales.
- Revenue trends highlighted changes in business performance over time.
- Sales varied across different cities and countries.
