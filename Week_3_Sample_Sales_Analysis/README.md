# 🎵 Chinook SQL Analysis

## 📌 Project Overview

This project was completed as part of the **AnalystLab Africa Data Analytics Internship (Week 3)**. The objective was to develop SQL querying skills by exploring and analysing the Chinook database, a relational music store database containing information about customers, employees, invoices, tracks, albums, artists, genres, playlists, and media types.

Using MySQL, I performed database exploration, wrote SQL queries to answer business questions, applied advanced SQL concepts, and improved query performance through indexing and query optimisation.

---

## 🎯 Objectives

- Explore and understand a relational database.
- Retrieve and analyse data using SQL.
- Apply advanced SQL concepts to solve business problems.
- Optimise SQL queries for improved performance.

---

## 🗂️ Database Structure

The Chinook database consists of multiple related tables, including:

- Customer
- Employee
- Invoice
- InvoiceLine
- Track
- Album
- Artist
- Genre
- Playlist
- PlaylistTrack
- MediaType

These tables are connected through primary and foreign keys, allowing customer purchases to be traced from invoices to the individual music tracks purchased.

## 📊 Business Questions Answered

This project answers several business questions, including:

- Who are the top-spending customers?
- Which tracks are purchased most frequently?
- Which artists generate the highest revenue?
- How has revenue changed over time?
- Which countries generate the most revenue?
- How do customer purchasing behaviours differ?
- Which tracks have never been purchased?
